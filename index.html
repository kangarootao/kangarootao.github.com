HoloLens+Unity Gaze

发表于 2017-05-23   |   分类于 HoloLens   |  

Gaze（凝视）是Hololens中用户用于确定全息目标的基本方法，用户视野中会出现一个随视角移动的光标用于选择物体。光标的产生是由用户的头向HoloLens发出射线，根据射线的碰撞位置显示。

在unity中，用户头的位置和方向和主相机一致，即UnityEngine.Camera.main.tranform.position与 UnityEngine.Camera.main.tranform.forward。随后即可通过物理射线Physics.RayCast导致的射线碰撞RaycastHit来确定碰撞到的物体是哪一个。
void Update()
{
RaycastHit hitInfo;
if (Physics.Raycast(
Camera.main.transform.position,
Camera.main.transform.forward,
out hitInfo,
20.0f,
Physics.DefaultRaycastLayers))
{
// If the Raycast has succeeded and hit a hologram
// hitInfo’s point represents the position being gazed at
// hitInfo’s collider GameObject represents the hologram being gazed at
}
}

关于gaze的可视化可以在GitHub的HoloToolkit-Unity仓库的GazeManager.cs中找到。Cursor.prefab和CursorWithFeedback.prefab展示了如何进行gaze光标的可视化。

HoloLens+Unity 坐标系统

发表于 2017-05-23   |   分类于 HoloLens   |  

HoloLens支持不同的体验范围（experience scales）。有orientation-only, seated-scale, room-scale, world-scale等几种。

Orientation-only or Seated-scale Experience


Namespace: UnityEngine.VR
Type: InputTracking
可以直接在Unity的世界坐标系中进行编辑，出现在相机视野里的物体将出现在应用中。

纯粹的orientation-only experience:
InputTracking.disablePositionalTracking = true;

seated-scale experience中重新调整seat位置：
InputTracking.Recenter();

Standing-scale or Room-scale Experience


Namespace: UnityEngine.VR.WSA
Type: StageRoot
第一次开启时检测地板与房间边界（墙壁）的位置，定义的物体将在此范围内。

要将Unity层次结构的分支定位到用户定义的层起始点，需将StageRoot组件添加到根游戏对象，随后这个对象将保持在指定的位置。
在脚本中可以通过调用StageRoot中的TryGetBounds方法获得边界多边形，随后即可进行碰撞检测。注意系统将在用户碰到时自动显示边界。

World-scale Experience


Namespace: UnityEngine.VR.WSA
Type: WorldAnchor

world-scale experience允许用户移动超过5米，这需要些额外的新技术。一种是空间锚点（spatial anchor）。不管用户走到哪，锚点会保存一系列的物理世界中全息图（holograms）以便后面使用。
在unity中，添加WorldAnchor组件给GameObject即可创建空间锚点。

添加WorldAnchor


WorldAnchor anchor = gameObject.AddComponent();

移除添加WorldAnchor


当前帧不移动
Destroy(gameObject.GetComponent());
当前帧移动
DestroyImmediate(gameObject.GetComponent());

移动锚点物体


DestroyImmediate(gameObject.GetComponent());
gameObject.transform.position = new Vector3(0, 0, 2);
WorldAnchor anchor = gameObject.AddComponent();

处理可定位变化


WorldAnchor在某个时间点可能无法在物理世界中找到。如果发生这种情况，Unity将不会更新锚点对象的变换。这也可以在应用程序运行时更改。不能处理可定位性的变化将导致物体不会出现在世界上正确的物理位置。需要订阅OnTrackingChanged事件，随后处理事件。OnTrackingChanged事件将在空间锚点可定位与不可定位切换时触发。
anchor.OnTrackingChanged += Anchor_OnTrackingChanged;
private void Anchor_OnTrackingChanged(WorldAnchor self, bool located)
{
// This simply activates/deactivates this object and all children when tracking changes
self.gameObject.SetActiveRecursively(located);
}
调用
Anchor_OnTrackingChanged(anchor, anchor.isLocated);

资料来源：https://developer.microsoft.com/en-us/windows/mixed-reality/Coordinate_systems_in_Unity.html#building_a_standing-scale_or_room-scale_experience

HoloLens+Unity 开发环境搭建

发表于 2017-05-23   |   分类于 HoloLens   |  

毕业设计选择了在HoloLens上进行应用开发，这里记录一下HoloLens的学习过程。目前打算制作基于HoloLens的三维中国象棋游戏，采用了Unity作为开发工具，毕竟微软提供了各种API接口供调用，应该算是最快速的开发方式之一了。

开发环境搭建


1.Visual Studio 2017 微软产品通用的开发工具。在Unity游戏完成后将其作为VS文件导出，再通过Visual Studio打开，编译并运行。
2.HoloLens Emulator HoloLens模拟器，其实就是虚拟机。需要Windows 10专业版或教育版才能运行，因为需要Hyper-V支持。Mac电脑可以装双系统的Windows使用模拟器，但是需要在Bootcamp中设置windows系统为首选系统开启才能使用，不然会检测到BIOS不支持虚拟机进而打不开模拟器。
3.Unity 5.5版本以上 游戏的主要开发环境，需要为HoloLens的使用进行配置。
Select File > Build Settings…
Select Windows Store in the Platform list and click Switch Platform
Set SDK to Universal 10
Set Target device to Any Device to support immersive headsets or switch to HoloLens
Set Build Type to D3D
Set UWP SDK to Latest installed

From the Build Settings… window, open Player Settings…
Select the Settings for Windows Store tab
Expand the Other Settings group
In the Rendering section, check the Virtual Reality Supported checkbox to add a new Virtual Reality Devices list and confirm “Windows Mixed Reality” is listed as a supported device (this will appear as “Windows Holographic” on some versions of Unity)

一些功能需要额外开启
Player Settings > Windows Store > Publishing Settings > Capabilities
SpatialPerception: SurfaceObserver
WebCam: PhotoCapture and VideoCapture
PicturesLibrary/VideosLibrary: PhotoCapture or VideoCapture, respectively (when storing the captured content)
Microphone: VideoCapture (when capturing audio), DictationRecognizer, GrammarRecognizer, and KeywordRecognizer
InternetClient: DictationRecognizer (and to use the Unity Profiler)

HoloLens的GPU仅是移动版的，需要降低图像质量来获得更高的速度
Select Edit > Project Settings > Quality
Select the dropdown under the Windows Store logo and select Fastest. You’ll know the setting is applied correctly when the box in the Windows Store column and Fastest row is green.

场景中的主相机需要进行设定
In the Inspector panel, set the transform position to 0, 0, 0 so the location of the users head starts at the Unity world origin.
Change Clear Flags to Solid Color.
Change the Background color to RGBA 0,0,0,0. Black renders as transparent in HoloLens.
Change Clipping Planes - Near to the HoloLens recommended 0.85 (meters).

一些额外的功能可以通过Unity APIs来实现，具体可以在Unity manual或Unity scripting reference中找到

资料来源：https://developer.microsoft.com/en-us/windows/mixed-reality/unity_development_overview

手游阴阳师分析

发表于 2017-04-24   |   分类于 游戏分析   |  

一、游戏类型
回合制策略类RPG，注重养成，手机游戏，日式和风背景，需联网游戏，可进行单人、多人异步、多人同步游戏。

二、游戏系统
1.等级
阴阳师等级：目前满级60，影响阴阳师属性以及剧情推进。提升方式：各类副本，结界突破（多人异步对战）。
御灵等级：目前满级40，影响阴阳师属性。提升方式：消耗御魂和金币。
阴阳师技能等级：影响技能有无以及技能效果。提升方式：任务，御灵等级。
式神：细分为星级、觉醒、式神等级、技能等级。
星级影响等级上限并大幅影响式神属性，二星满级20，每级提升5上限，最高六星满级40。提升方式：消耗和星级数数量以及星级相等的式神和金币。1六星=1满级五星+5五星=6满级四星+24四星=30满级三星+90三星=120满级二星+240二星。一满级六星所需经验值：（30-40）1+（25-30）6+（20-25）30+（1-20）120。培养困难，大后期以外玩家的目标。
觉醒影响式神属性，式神技能。提升方式：消耗觉醒材料和金币。根据式神稀有度不同消耗的种类与数量也不相同。培养相对简单，为前期玩家的目标。
式神等级影响式神属性。提升方式：各类副本、结界突破、挂机养成、消耗式神和金币。
技能等级影响技能效果。提升方式：消耗同种式神或黑蛋随机提升。
御魂等级：目前满级15，影响御魂主属性以及新增副属性。提升方式：消耗御魂和金币。

2.职业
可选阴阳师有四个，可随时替换。共享等级、不共享御灵。

3.能力值
基础属性有攻击、生命、防御、速度、暴击、暴击伤害、效果命中、效果抵抗。

4.战斗
回合制策略战斗，每个式神和阴阳师有着各自的回合，在回合内玩家可选择该回合使用的技能与目标。可选择手动或自动战斗，可调一倍速与二倍速。战斗分为三式神战与五式神战。个人战斗、同步对战、同步协同作战。
战斗流程：进度条模式，当前没有人物进度抵达指定数值时根据时间计数x，人物进度=速度*x。当人物进度达到指定数值（进度条终点）时停止计数，进行此人物的回合，结束回合后进度归零。回合内玩家选择技能与目标，进行攻击、加buff、上debuff、治疗、复活等一系列操作。直至某一方全灭则战斗结束。战斗资源为魂火，初始为3，每五次行动获得3、4、5点魂火，可以由式神技能或御魂技能获得，某些技能会消耗魂火。每个式神的站位固定，每方有且仅有一个固定的召唤物位置。

5.剧情
目前出了20章剧情，剧情推进通过场景对话框，场景内动画配字幕（试验性加入弹幕），战斗。总体剧情较短，一个简单的主线和体现式神性格与关系的几个支线，并不出彩。

6.角色
即式神，游戏的核心。获得方式有抽卡，碎片合成。可以更改式神名字、皮肤。通过传记与图鉴简介了解式神背景。在式神界面可以查看其它玩家的评价。式神绘卷可以一次性展示所获得式神并可进行分享。式神拥有2d立绘、3d模型、日语配音（点击一句，介绍一句，技能三句，挨打一句，传记三段）。

7.装备
即御魂。获得方式有商店购买、御魂副本（主要方式）、boss战、普通副本等。分星级与种类。不同星级的属性有差异，不同种类的御魂技能有差异。御魂技能有4件套与2件套加成，一般选用4+2的搭配方式。御魂拥有主要属性与次要属性，主要属性随等级提升而提升，次要属性随等级提升缓慢增加（可能会产生新的属性）。由于掉落的御魂属性随机，而不同位置的主要属性偏向也不同，所以需要大量获得来筛选出最优的属性，成为游戏后期遥远的目标。

8.物品
符咒券、皮肤券、挑战券、觉醒材料、式神碎片等。拥有各自的用途。

9.技能
阴阳师技能：阴阳师拥有7个技能，每次获得技能点会随机加到某一个技能或普攻上。可以选择其中的两种进行战斗。技能有冷却时间。
式神技能：一般有2个技能，每次活动技能点会随机加到某一个技能或普攻上。一般会消耗魂火，没有冷却时间。

10.经济
没有直接的玩家交易。一级货币勾玉，二级货币金币、友情点、荣誉点、勋章、体力。
勾玉：可以购买体力、金币、抽卡券、经验加成。获得方式：充值、每日任务、每日活动、副本首通、大型活动等。
金币：消耗式神、觉醒、升星、消耗御魂等功能需要消耗。也可以用于购买御魂（限定，需用勾玉刷新或等自动刷新）。
友情点：可以购买小抽卡券、经验式神、升星式神（周数量限制）。获得方式：每日好友互赠。
荣誉点：可以购买抽卡券、经验式神、某ssr碎片。获得方式：同步玩家对战获得（无论输赢）。
勋章：可以购买很多物品（周数量限制）。获得方式：异步玩家对战、每日活动、阴阳寮赠予。
体力：用于打副本。获得方式：自然恢复、勾玉兑换、时间段奖励、任务奖励。
间接玩家交易：阴阳寮祈愿，给某成员他所指定的式神碎片，有上限。

11.商店
游戏仅有一个商店，所有的玩家与系统交易都在此产生。包括皮肤兑换以及各式各样的礼包。限时单次礼包、每日单次礼包、每几日限定礼包（月卡）、周末礼包。

12.任务
成就任务：阶段性成果奖励，奖励丰厚。
每日任务：各种玩法总结任务，只有全完成的奖励较好。
悬赏封印：每日两次刷新，打一定数量随机指定的式神，奖励较好。有概率刷出邀请好友完成的合作任务，奖励更好。

13.成就
各种阶段性成就任务，完成各类副本boss的首杀、首次获得某类物品、等级到达某个值、还有隐藏的X级非酋等。

14.副本
普通副本、困难副本、觉醒副本、御魂副本、每日定时副本、秘闻副本（高难度、特殊机制）、大型boss副本和式神碎片副本（普通及困难副本打完后随机出现）。不同副本的战斗人数有区别，部分可与他人组队完成。普通副本中可自由选择攻略的顺序，并且怪物奖励在进副本时随机生成，副本完成后奖励随机度较高、可能出现大型boss副本、式神碎片副本、经验副本等，不可预知性高。

15.竞技
分为同步玩家对战与异步玩家对战。异步玩家对战对方为设定好的阵容，自动战斗。同步玩家对战仅在限定时间段开展，随着回合数增加双方伤害会逐渐增加，加快游戏节奏。

16.匹配
游戏仅有对战匹配。匹配基准为斗技点数，应该是选择同时匹配的最接近点数进行配对。双方匹配点数将影响输赢的点数获取以及减少。

17.社交
游戏内社交：
好友系统，可聊天、切磋、查看空间（可放照片，写qq号微信号，简介等）、式神寄养、互赠友情点、组队、协同任务等。好友羁绊上升可升级为亲友，可进行异步式神支援，额外组队加成，上线提醒等。
阴阳寮，即工会。每日可祈愿等其他人赠送碎片，会长副会长可以购买增益效果和礼包（消耗阴阳寮勋章），可开启集体玩法（狩猎战，妖怪退治等，即boss战）。地图玩法可显示周围的玩家以及阴阳寮，以及进行狩猎战。
游戏外社交：
内置链接引向官方游戏平台，有社区讨论、同人作品、游戏直播等。线下参与各种漫展、游戏展提高知名度，与各单位联动如KFC、农夫山泉。线上开展同人作品比赛、MMD大赛等。

18.外观
游戏可更换庭院（主界面），式神（皮肤），阴阳师（皮肤）的外观。只影响外观不影响游戏内容。

19.排名
游戏仅有斗技排名。以斗技点数为主键排序。

三、美术风格
角色风格：介于Q版与写实版之间，风格统一。2d原画朴素，3d模型制作在手游中较好，骨骼动作制作用心。不同角色动作各不相同，战斗中额外动作不少（如转头看），面部表情无变化。
效果：风格统一。不同角色效果不一，中规中矩。
场景：场景原画分层抠图插片摆放，战斗场景2d+3d混合，还原原画质量。
UI：同样风格统一，制作精良。

四、音乐与音效
音乐：梅林茂配乐，平安时期和风音乐。符合游戏背景，平静之中带有些许阴冷，然而无法给人以深刻印象。
音效：中规中矩。

五、评价
优点：
1.美术优质且一致。从角色到场景到UI都属于高水准，领先同时期的其他手游。在二次元与现实画风之间找到平衡，吸引了两方面的玩家。
2.手机操作回合制战斗简单，适合大多数玩家的战斗难度与深度。
3.给予玩家一些遥不可及的目标（御魂随机属性），延长游戏寿命。
4.游戏内社交功能完善。
5.运用了一些较新的技术吸引玩家，如AR召唤、现实地图社交等。
6.官方积极开展联动和同人活动。

缺点：
1.游戏内容缺乏，玩法多样性低。PVE单一且没有变数，PVP限制时间。
2.在游戏内容缺乏的基础上绑定玩家大量时间，导致厌烦。
3.数值线单一，最终区分都在御魂与式神上。
4.角色塑造薄弱，语音太少，剧情太少。

被引导的非主流的审美体系

剧情：非必要条件，但是是RPG类游戏的主要组成部分。

2.职业系统

发表于 2017-03-28   |   分类于 游戏系统   |  

系统目标
1.对玩家或角色的职能进行区分，不同职业拥有的功能不同，例如输出、坦克、治疗等。
2.提高角色多样性，不同职业以及搭配能给予多样的新鲜感。
3.可以设置职业克制，在一定条件下筛选出“最优”选择。

实现方法
1.设置不同职业供玩家选择。
2.达到要求时可进行下一步职业进阶（二转）。
3.特定条件下可更换职业。
4.特定游戏中可以同时拥有多职业。

种类
按功能分类：
输出：
1.根据攻击距离不同，分为远程输出与近战输出。
2.根据输出方式不同，分为物理输出与魔法输出。
3.根据输出属性不同，分为各类属性攻击，如火、冰、雷。
4.根据输出时能否兼顾走位，分为站桩输出与移动输出。
5.根据输出目标的多少，分为单体输出与群体输出。
6.根据输出的生效时间，分为瞬时输出与滞后输出。

治疗：
1.根据治疗生效时间，分为即时治疗、持续治疗与预先治疗。
2.根据治疗目标的多少，分为单体治疗与群体治疗。
3.根据治疗时能否兼顾走位，分为站桩治疗与移动治疗。

坦克：
1.根据防御方式不同，分为物抗坦克、魔抗坦克、闪避坦克（还有格挡、招架等）。
2.根据能否自我治疗，分为自疗坦克与非自疗坦克。
3.根据属性倾向不同，分为护甲型（高减伤）与血量型（高血量）。

辅助：
1.根据辅助技能的性能不同，分为增强辅助与削弱辅助。
2.根据目标数量不同，分为自身、单体、群体辅助。
3.根据辅助技能种类不同，分为硬控（受控者无法操作，如晕眩、睡眠）、软控（受控者可以操作但受限，如减速、致盲）、削弱（降低属性）、战斗增益（增强属性）、非战斗增益（传送、造食）。

职业倾向可以由以上不同的功能进行组合后得出。以WoW为例，冰法属于远程、魔法、冰系、半站桩半移动、单体倾向的爆发输出，同时拥有预先治疗手段（冰盾），擅长对敌方单体与群体进行硬控与软控（深度冻结与冰霜新星）。以WOT为例，重坦一般属于较近战、站桩、单体瞬时输出，兼具护甲型坦克的属性。

1.等级系统

发表于 2017-03-28   |   分类于 游戏系统   |  

系统目标
1.量化玩家成长
对于大目标（满级）进行分解，分成各个阶段性小阶段，让玩家总能够找到一个就快要完成的小目标，完成后再开放下一个，促使玩家继续努力。并且在进行一段时间游戏后能够从数值上了解自己在哪些方面成长了，成长了多少，进而达到付出就有收获的满足感。

2.直观的实力区分
对于各类等级（玩家等级，装备等级，随从等级等）进行加权求和可以得出战斗力（装备分），能够作为玩家的实力证明之一。在进行玩家对战或者组队时可以作为事前评估。同样也可以作为炫耀的资本之一。

3.限制游戏内容
同样作为目标之一，常见于随等级开放的主线任务与剧情，或是装备的等级要求。目标明确的情况下，为了达到下一阶段玩家同样会付出努力进行等级的提升。

实现方法
1.完成对战
最常见的获取方式。通过完成对敌方的击杀、副本的通过、与玩家的对战获取经验值。而玩家在此过程中不断重复着练级的过程，体验游戏的主要内容。一般作为单机，端游的主要经验值获取形式。

2.任务获取
任务作为一种明确目标，在对战之余给予玩家第二种等级提升渠道。一般在对战，探索的基础上加入了剧情与物品奖励，且性价比较高。每日任务可作为休闲玩家的主要等级提升渠道，同时也能提高用户粘性。

3.使用物品
使用经验狗粮一类的物品进行玩家、随从的等级提升，使用金钱以及材料进行装备、物品的等级提升。相比于对战与任务更加快速，省事。一般作为手游、页游的主要经验值获取形式。

种类：
1.玩家等级
RPG游戏中作为实力的数值体现。随着等级的提高，人物属性、技能也将随之提高，实力逐步增强。
玩家对战游戏中作为作为实力或资历的体现。由于玩家对战需要公平，一般只会轻微或者不影响实力，但是等级高低能够反应玩家投入的时间。
卡牌手游中一般作为资历的体现。一般只会影响主线的推进或者体力的上限。

2.装备等级
一方面是装备的本身具有的属性带来的“等级”区分。越稀有的装备属性越高，因此也常用颜色区分装备的属性等级（也可以说是稀有度）。
另一方面是针对特定装备进行的强化、附加物品（宝石）等。因为可以设置为难以提升且不影响游戏的主体体验，常作为游戏后期的主要目标之一。

3.技能等级
通过技能等级的成长提高技能的伤害、附加效果、减少冷却时间等。生活技能的成长能允许玩家制造或生产更多的材料。可以通过限制技能数量、技能点的获取迫使玩家必须做出抉择，需要加哪一些技能、加几点、哪个先加。游戏可玩性的重要组成部分之一。

4.角色等级
卡牌游戏的重要组成部分，也就是核心的养成部分。如暗黑系列的随从、舰队Collection的老婆们、Fate Grand Order的Servant等。可以进一步细分为角色本身等级，角色羁绊等级。

5.物品等级
可以用来衡量物品的使用属性、稀有度等，例如各种等级的回复药、属性宝石等。

6.职业等级
一般与玩家等级绑定，但极少数游戏中与玩家等级分离（如RO仙境传说）。设置两种不同的等级成长线好处在于转职后的技能获取更加迅速。
